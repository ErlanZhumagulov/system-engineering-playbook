# Реестр рисков — LMS+P

---

## Матрица приоритизации рисков

Оценка по двум осям: **Вероятность** (В) и **Ущерб** (У) — по шкале 1–3 (1=низкий, 2=средний, 3=высокий/критический).
**Приоритет** = В × У. Чем выше — тем критичнее.

| № | Риск | Вероятность | Ущерб | Приоритет | Меры снижают до |
|---|---|:---:|:---:|:---:|:---:|

| R-02 | Высокий FPR детектора нарушений | 3 | 3 | **9** | Низкий |
| R-01 | Отказ медиасервера во время экзамена | 2 | 3 | **6** | Средний |
| R-04 | Потеря ответов студента при разрыве | 2 | 3 | **6** | Очень низкий |
| R-07 | Мошенничество через удалённое управление | 2 | 3 | **6** | Средний |
| R-06 | Сбой передачи оценок в SIS | 2 | 2 | **4** | Низкий |
| R-10 | Деградация детекции при плохой камере | 2 | 2 | **4** | Низкий |

## Детальный реестр рисков

---

### R-01. Отказ медиасервера во время экзамена

| Поле | Описание |
|---|---|
| **Причина** | Технический сбой медиасервера, перегрузка при пиковой нагрузке, сетевой разрыв между LMS+P и медиасервером |
| **Событие** | WebRTC-соединение разрывается; студент теряет возможность передавать видеопоток |
| **Последствие** | Экзамен прерывается; прокторинг в реальном времени невозможен; студент несёт психологическую нагрузку |
| **Вероятность** | Средняя (2) |
| **Ущерб** | Высокий (3) — прямое влияние на ход академического экзамена |
| **Приоритет** | 6 |
| **Меры** | Автопереключение в асинхронный режим (локальная запись в браузере); таймер сессии паузируется при разрыве >30 сек; горизонтальное масштабирование медиасервера (K8s KEDA); circuit breaker в Proctoring Service |
| **Остаточный риск** | Средний — асинхронный режим не обеспечивает наблюдения в реальном времени |

---

### R-02. Высокий уровень ложноположительных срабатываний детектора нарушений

| Поле | Описание |
|---|---|
| **Причина** | Плохое освещение, очки, маска, движение вентилятора, фоновый шум детектируются как нарушение |
| **Событие** | Студенты без нарушений получают тревоги; проктор перегружен ложными событиями |
| **Последствие** | Несправедливые вердикты; жалобы студентов; потеря доверия к системе |
| **Вероятность** | Высокая (3) — особенно при разнородных домашних условиях |
| **Ущерб** | Высокий (3) — репутационный и академический ущерб |
| **Приоритет** | 9 |
| **Меры** | Confidence score + настраиваемый порог; обязательная проверка проктором перед финальным вердиктом; предэкзаменационный camera check; feedback loop для калибровки модели; мониторинг метрики FPR (NFR-05) |
| **Остаточный риск** | Низкий — FPR ≤ 15% как измеримое NFR; проктор принимает финальное решение |

---

### R-03. Недоступность SSO/IdP при проведении экзаменов

| Поле | Описание |
|---|---|
| **Причина** | Технический сбой корпоративного IdP (Keycloak, Azure AD) |
| **Событие** | Студенты не могут войти в систему; новые сессии не создаются |
| **Последствие** | Экзамены срываются; требуется перенос; организационный и репутационный ущерб |
| **Вероятность** | Низкая (1) — IdP — критическая инфраструктура организации |
| **Ущерб** | Критический (3) |
| **Приоритет** | 3 |
| **Меры** | JWT с TTL 8 ч — активные сессии не прерываются; мониторинг доступности IdP с алертом заблаговременно; инструкция аварийного входа для администратора; рекомендация: IdP с HA-конфигурацией |
| **Остаточный риск** | Низкий — активные сессии продолжаются; блокируются только новые входы |

---

### R-04. Потеря ответов студента при разрыве соединения

| Поле | Описание |
|---|---|
| **Причина** | Нестабильный интернет у студента, сбой браузера, разряд устройства |
| **Событие** | Студент теряет связь; несохранённые ответы могут быть утеряны |
| **Последствие** | Частичная потеря ответов → несправедливый результат; жалобы и апелляции |
| **Вероятность** | Средняя (2) |
| **Ущерб** | Высокий (3) |
| **Приоритет** | 6 |
| **Меры** | Автосохранение на сервере при каждом переходе между вопросами (SYS-09); `Idempotency-Key` для предотвращения дублей; localStorage как кэш ответов на стороне клиента; синхронизация при восстановлении соединения |
| **Остаточный риск** | Очень низкий — потеря возможна только для текущего вопроса, если переход ещё не произошёл |

---

### R-05. Несанкционированный доступ к видеозаписям экзаменов

| Поле | Описание |
|---|---|
| **Причина** | Компрометация API-ключа S3; утечка pre-signed URL; ошибка в политиках доступа S3 |
| **Событие** | Посторонние лица получают доступ к видеозаписям без соответствующих прав |
| **Последствие** | Нарушение приватности студентов; ответственность по GDPR/152-ФЗ; штрафы; репутационный ущерб |
| **Вероятность** | Низкая (1) |
| **Ущерб** | Критический (3) |
| **Приоритет** | 3 |
| **Меры** | Pre-signed URL с TTL 4 ч (SYS-29); аудит-лог всех обращений (SYS-31); ротация S3-ключей; шифрование AES-256 (SYS-29); регулярный аудит политик доступа S3; rate limiting на endpoint получения URL |
| **Остаточный риск** | Низкий |

---

### R-06. Сбой передачи оценок в SIS

| Поле | Описание |
|---|---|
| **Причина** | Плановые работы на SIS; изменение API SIS без уведомления; сетевые проблемы |
| **Событие** | Оценки из LMS+P не поступают в деканат |
| **Последствие** | Задержка в академической документации; студенты не видят официальных оценок в деканате |
| **Вероятность** | Средняя (2) |
| **Ущерб** | Средний (2) — оценки не теряются, только задерживаются |
| **Приоритет** | 4 |
| **Меры** | Очередь с retry (1/5/15/60 мин) (SYS-25); алерт администратору после 5 неудач; ручная повторная отправка из UI; мониторинг статуса интеграции на дашборде |
| **Остаточный риск** | Низкий — данные не теряются, доставка задерживается |

---

### R-07. Академическое мошенничество через удалённое управление устройством

| Поле | Описание |
|---|---|
| **Причина** | Студент использует TeamViewer, AnyDesk или аналоги во время экзамена |
| **Событие** | Третья сторона управляет компьютером студента без обнаружения |
| **Последствие** | Нечестный результат экзамена; несправедливость по отношению к другим студентам |
| **Вероятность** | Средняя (2) |
| **Ущерб** | Высокий (3) |
| **Приоритет** | 6 |
| **Меры** | Опциональный античит-агент (lockdown browser) блокирует сторонние приложения; детекция смены активного окна (SYS-17); анализ поведенческих паттернов (скорость ввода — опционально) |
| **Остаточный риск** | Средний — без lockdown browser полная защита невозможна |

---

### R-08. Перегрузка системы в начале экзаменационной сессии

| Поле | Описание |
|---|---|
| **Причина** | Все студенты одновременно входят и запускают сессии в первые 5 минут экзамена |
| **Событие** | Резкий spike нагрузки; API Gateway, Proctoring Service и медиасервер перегружены |
| **Последствие** | Задержки и отказы в запуске сессий; студенты теряют время экзамена; жалобы и апелляции |
| **Вероятность** | Высокая (3) — предсказуемый паттерн нагрузки |
| **Ущерб** | Высокий (3) |
| **Приоритет** | 9 |
| **Меры** | K8s HPA — автомасштабирование по CPU/RPS (NFR-06); предварительное прогревание пула воркеров медиасервера; rate limiting (SYS-33); нагрузочное тестирование перед каждой сессией; рекомендация: разнесение окон начала экзаменов по 5–10 мин |
| **Остаточный риск** | Низкий — при правильно настроенном HPA и прогреве воркеров |

---

### R-09. Несоответствие требованиям GDPR при хранении видеозаписей

| Поле | Описание |
|---|---|
| **Причина** | Видеозаписи хранятся дольше установленного срока или используются без согласия студента |
| **Событие** | Нарушение политики хранения персональных данных |
| **Последствие** | Штрафы по GDPR (до 4% годового оборота); судебные иски студентов; репутационный ущерб |
| **Вероятность** | Низкая (1) |
| **Ущерб** | Критический (3) |
| **Приоритет** | 3 |
| **Меры** | Автоматическое удаление по TTL (S3 lifecycle policy, SYS-32); согласие студента перед экзаменом (SYS-02); аудит удалений; право на удаление данных (SYS-30); хранение в согласованной юрисдикции |
| **Остаточный риск** | Очень низкий |

---

### R-10. Деградация качества детекции при низком качестве камеры студента

| Поле | Описание |
|---|---|
| **Причина** | Студент использует веб-камеру с низким разрешением или находится при плохом освещении |
| **Событие** | Proctoring Service не может корректно обнаружить лицо или события нарушений |
| **Последствие** | Снижение достоверности прокторинга; повышение FPR или пропуск нарушений |
| **Вероятность** | Средняя (2) |
| **Ущерб** | Средний (2) |
| **Приоритет** | 4 |
| **Меры** | Предэкзаменационный camera check с оценкой качества изображения; минимальные требования к оборудованию в интерфейсе; fallback: рекомендация улучшить освещение; возможность fallback на асинхронный режим |
| **Остаточный риск** | Низкий |
